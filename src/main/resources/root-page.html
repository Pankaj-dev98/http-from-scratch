HTTP From Scratch

A handwritten HTTP/1.1 server built directly on top of TCP sockets in Java. It parses raw request bytes, validates request structure, routes requests without a web framework, and writes well-formed responses back to the client.

The project also demonstrates upstream integration through Java's HttpClient, streaming proxy responses with Transfer-Encoding: chunked, and trailer metadata for response integrity.

Technical Highlights

  • Raw HTTP parsing: request line, headers, and body are parsed from the socket stream without a framework.
  • Bounded parsing: request growth is capped with an 8 KB max header budget and a 100-header limit.
  • Protocol validation: malformed request lines, invalid headers, and mismatched content length are rejected.
  • Streaming proxy: proxied upstream responses are forwarded incrementally instead of being buffered in memory.
  • Flexible responses: the server emits text/html, text/plain, and upstream content types such as application/json.

Project Surface

/funfact Shows outbound API usage and dynamic response generation. /httpbingo/stream/100 Demonstrates proxying, chunk boundaries, trailer emission, and back-pressure-friendly streaming.
Implementation note: the server handles each connection on a virtual thread and can be deployed with a standard PORT environment variable, which makes it suitable for container-based hosting platforms and portfolio demos.
